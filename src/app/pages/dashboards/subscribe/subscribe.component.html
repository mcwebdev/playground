Observables are a fundamental concept especially in libraries such as RxJS for handling asynchronous tasks and events. Here are some keywords and concepts associated with Observables:

Observable:

A representation of any set of values defined over time.

Observer:

An interface for a consumer of an Observable that receives values.

Subscription:

Represents the execution of an Observable.

Operators:

Functions that build on the observables foundation to enable sophisticated manipulation of collections.

Subject:

A special type of Observable that allows values to be multicasted to many Observers.

Scheduler:

A central dispatcher for scheduling tasks, allowing control when execution happens.

Hot Observable:

An Observable that begins emitting items as soon as it is created.

Cold Observable:

An Observable that begins emitting items when it is subscribed to.

next:

A method of the Observer interface, called to deliver the next value from the Observable.

error:

A method of the Observer interface, called to deliver an error notification from the Observable.

complete:

A method of the Observer interface, called to deliver a complete notification from the Observable.

create:

A method used to create a new Observable.

pipe:

A method used to compose Operators together.
